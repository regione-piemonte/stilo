Implementación del framework de validadores de confianza de MITyCLibAPI para política de confianza del MITyC.

Se implementa una solución propia basada en 4 tipos de validadores de confianza diferenciados que implementan los interfaces:

  • {@link es.mityc.javasign.trust.ITrustSignCerts}: validador de confianza de cadenas de certificados.s
  • {@link es.mityc.javasign.trust.ITrustCRLEmisor}: validador de confianza de CRLs.
  • {@link es.mityc.javasign.trust.ITrustOCSPProducer}: validador de confianza de respuestas OCSP.
  • {@link es.mityc.javasign.trust.ITrustTSProducer}: validador de confianza de sellos de tiempo.
Se implementa una factoría propia ({@link es.mityc.javasign.trust.TrustExtendFactory}) que instancia los validadores según el tipo requerido.

La nueva factoría utiliza los mismo ficheros de configuración que la antigua factoría (trust/trust.properties) para asociar claves con managers de confianza aplicando un formato de clave específico. Según el tipo de validador buscado se incluye un sufijo en la clave:

  • .SignCerts: para validador de cadenas de certificados
  • .CRLEmisor: para validador de CRLs
  • .OCSPProducer: para validador de respuestas OCSP
  • .TSProducer: para validador de sellos de tiempo TSA
Ejemplo de uso: 
            (((TrustExtendFactory)TrustFactory.getInstance()).getSignCertsTruster("mityc"));
donde hay un fichero de propiedades con el valor 
            mityc.SignCerts=<nombre de la clase del manager de confianza de cadenas de certificados que implemeta el interfaz ITrustSignCerts>

El paquete incluye una implementación propia de managers de confianza basados en ficheros de propiedades. Estos ficheros de propiedades indican los recursos que contienen los certificados en los que se confía (como emisores directos o como issuers de los emisores).
